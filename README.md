Project theme: **AI vs Human Content Detection** (text classification).

## Labs Overview

### Lab 2: Identifying ML Problems, Selecting Open Datasets, and Drawing a Methodology Diagram

#### 1) Dataset Selection
- **Dataset:** `ai_human_detection_v1.csv`
- **Source:** Self-curated / Kaggle-style dataset
- **Type:** NLP + metadata (tabular dataset containing text and related fields)
- **Original Size:** ~686 rows, 11 columns (may include noisy error rows)
- **Text Types:** essays, emails, blog posts, social media-style posts
- **Languages:** includes English and Spanish (and may contain other languages)

#### 2) Problem Definition
- **Problem Type:** Supervised classification
- **Goal:** Predict whether a text sample was written by a **Human** or generated by an **AI**
- **Target column:** `human_or_ai`

**Labeling used in the project**
- The dataset contains: `human`, `ai`, and `post_edited_ai`
- For binary experiments:  
  - **Human = 0**  
  - **AI = 1** (where `ai` and `post_edited_ai` are grouped together)

**Input features**
- Primary feature: **`text`**
- Metadata fields (e.g., `domain`, `language`, `word_count`, `generation_date`, `prompt`) are used mainly for analysis/EDA.
- Note: some metadata can cause label leakage (e.g., `source_model`, `prompt` missingness), so the main baseline model is **text-only**.

---

### Lab 3: Data Understanding and Exploration (EDA)

#### Key Findings
Exploratory analysis shows clear differences between classes:

- **Length:** AI-generated texts tend to be longer on average than human texts.
- **Vocabulary richness:** Human writing often shows higher lexical diversity.
- **Structure:** AI texts frequently appear more consistently structured and “polished.”
- **Noise discovered:** Some rows contain API error messages (e.g., “Client Error”), which must be removed before modeling.

#### EDA Outputs
- Class distribution plots
- Word-count and character-length distributions by label
- Basic text statistics (e.g., approximate word count, character count)

---

### Lab 4: Data Quality & Preprocessing

This lab focuses on improving data quality and preparing numeric features for analysis:

#### Task 1 — Data Quality Issues
Identified issues:
- Missing values (mostly in `prompt`)
- Duplicate rows (if present)
- Noisy/error text rows (API errors)
- Class imbalance

#### Task 2 — Missing Value Strategy
- Strategy applied: Fill missing `prompt` with `"Unknown"`  
- Reason: keeps dataset consistent and avoids dropping rows due to missing categorical metadata.

#### Task 3 — Outlier Detection (IQR)
- Numeric features extracted from text (e.g., char length, word length, punctuation count).
- Outliers detected using the **IQR method**.
- Outliers handled using **capping (Winsorization)** to IQR bounds.

#### Task 4 — Normalization
Applied both:
- **Min-Max scaling**
- **Z-score standardization**

#### Task 5 — PCA
- PCA applied on standardized numeric features.
- Explained variance and cumulative variance were plotted to interpret dimensionality reduction.
